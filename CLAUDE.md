# Sunny Pint

Find sunny beer garden seats. Real-time shadow maps for UK pubs using LiDAR elevation data, OSM building outlines, and geometric sun projection.

**Domain**: sunny-pint.co.uk

## Status: UK Live

38k pubs (33k England + 2.4k Scotland + 2.5k Wales). All pubs have LiDAR heights, terrain horizons (3km range), outdoor areas, and Sunny Ratings. Data served from Cloudflare R2 via `data.sunny-pint.co.uk`.

## Quick Start

Runs in a devcontainer. Open in VS Code with Dev Containers extension.

```bash
pnpm install          # Frontend dependencies
just dev              # Vite dev server (port 5173)
just release          # Production build + SEO landing pages → dist/
```

## Data Pipeline (v2)

Data files are NOT in the repo — they're generated by the pipeline and served from R2.

```bash
uv run python pipeline/run.py --area uk          # Full pipeline
uv run python pipeline/run.py --area uk --dry-run # Preview what would run
uv run python pipeline/run.py --area uk --force   # Ignore manifest, re-run all
uv run python pipeline/run.py --area uk --stage enrich,package  # Specific stages
```

The pipeline auto-downloads all data sources on first run:
- OSM extracts from Geofabrik (~1.5 GB)
- OS Terrain 50 from OS Open Data API (155 MB)
- LiDAR DTM/DSM from Defra survey catalogue (streamed per-tile)
- INSPIRE parcels from HM Land Registry + Registers of Scotland

### Stages

```
Stage 1: EXTRACT     OSM .pbf → pubs_merged.json + buildings.gpkg
Stage 2: INDEX       INSPIRE GMLs → inspire.gpkg + scotland_parcels.gpkg
Stage 3: ENRICH      LiDAR + parcels → pubs_enriched.json (heights, horizons, outdoor areas)
Stage 4: PACKAGE     pubs_enriched → pubs.json + pubs-index.json + per-pub files + buildings.pmtiles
Stage 5: SCORE       pubs.json → sun scores → regenerates index + per-pub files
```

Independent horizon recompute (no re-download of LiDAR):
```bash
uv run python pipeline/stages/horizon.py --area uk          # Add/update horizon_dist
uv run python pipeline/stages/horizon.py --area uk --force  # Recompute all
```

### Deploy

```bash
just deploy-data      # Upload index + chunks + PMTiles + OG cards to R2 (boto3, ~67/s)
```

Output: `public/data/pubs-index.json` (R2), `public/data/pub/{slug}.json` (R2), `public/data/buildings.pmtiles` (R2).

See `pipeline/DESIGN.md` for full architecture.

## Architecture

**Static site** — no backend at runtime. All shadow computation is client-side at 60fps.

### Frontend (`src/`)
- **Vite 8** + TypeScript 6 + Tailwind 4
- **Canvas 2D** — porthole renderer, sun arc widget
- **SunCalc** — sun position
- **Lucide** — icons
- **VitePWA** — offline/installable

Key modules (see `.claude/rules/frontend.md` for full list):
- `circle.ts` — porthole canvas (buildings, shadows, terrain shadows, bezel, compass, pub sign)
- `sunarc.ts` — sun arc time picker with play animation
- `shadow.ts` — geometric shadow projection + terrain shadow edge computation
- `buildings.ts` — building data loader (PMTiles range requests from R2)
- `publist.ts` — pub list with search and distance sorting
- `state.ts` — app state, pub selection, time

### Data Pipeline (`pipeline/`)
- `run.py` — orchestrator with manifest-based change detection
- `stages/extract.py` — OSM .pbf → pubs + buildings (single osmium pass)
- `stages/index.py` — INSPIRE GMLs → spatial GeoPackage
- `stages/enrich.py` — LiDAR heights + horizons + parcel matching + outdoor areas
- `stages/package.py` — town/country/slug derivation, index/chunk splitting, PMTiles
- `stages/score.py` — sun scoring (shells out to precompute_sun.ts), regenerates splits
- `stages/horizon.py` — independent horizon recompute with OS Terrain 50 (3km range)
- `utils/terrain50.py` — OS Terrain 50 loader (50m DTM, all GB)
- `utils/bundles.py` — Defra LiDAR bundle download/decode
- `utils/download.py` — auto-download OSM extracts + OS Terrain 50

Legacy v1 scripts remain in `scripts/` for reference but are not used by the pipeline.

### Data Serving
- **Cloudflare R2** — bucket `sunny-pint-data`, domain `data.sunny-pint.co.uk`
  - `pubs-index.json` (~12.6 MB / ~2 MB gzip) — slim pub list (id, slug, name, lat, lng, town, county, country, sun, opening_hours, outdoor_area_m2) loaded at SPA startup for list rendering, search, distance sort, open-now filter
  - `pub/{slug}.json` (~3 KB each, 38k files) — full per-pub record (everything in slim + outdoor polygon, elev, horizon, BarOrPub schema fields) plus a pre-computed `nearby` array of the 10 closest pubs. Fetched directly by the `/pub/[slug]` Pages Function (no index parse on cold start) and lazy-loaded by the SPA on pub selection.
  - `buildings.pmtiles` — building vector tiles, range-requested by pmtiles.js
  - `og/{slug}.jpg` — pre-rendered OG card images
- **Cloudflare Pages** — `dist/` contains only the SPA shell (HTML/JS/CSS/icons), zero data files
  - `just release` strips all data from dist/ before deploy
  - `just deploy-data` uploads data to R2 via boto3 S3 API

### Deployment
- **Cloudflare Pages** — auto-deploys on git push to main
- Per-pub HTML pages rendered on-demand by Pages Functions (`functions/pub/[slug].ts`)
- OG images: pre-rendered as static JPEGs on R2
- Static city/theme landing pages generated at build time by `generate_pages.ts`
- Pages Functions fetch pub data from R2 (not from dist/)

## Critical Technical Context

- Building heights use DSM minus DTM — 90th percentile within footprint captures ridge height
- Shadow projection uses actual building height only — elevation differences are NOT added (terrain shadow handles hillside occlusion)
- Terrain shadow edge: computed from horizon angle + ridge distance, rendered as a half-plane sweeping across the porthole
- Horizon profiles: 36 azimuths, 1m DTM for 0-500m detail, OS Terrain 50 for 500-3000m distant ridges
- Shadow quads from overlapping buildings need offscreen canvas compositing to prevent opacity stacking
- Shadow max length capped at 200m to avoid infinite geometry near sunrise/sunset
- Height-dependent building filter: short buildings excluded if too far to cast useful shadows
- Day/night transition uses `dayFrac = (altitude + 2) / 10` for smooth 30-minute twilight
- Mapbox tiles: streets-v12 + satellite-streets-v12 (public token, URL-restricted to sunny-pint.co.uk)
- Porthole uses tile zoom 18 metres-per-pixel for building overlay alignment
- LiDAR sources: EA/Defra bundles (England), JNCC WCS (Scotland), NRW COG (Wales)
- OS Terrain 50: 50m DTM covering all GB, used for long-range horizon rays (auto-downloaded, 155MB)
- Outdoor area polygons support holes (evenodd canvas fill) for enclosed buildings
- precompute_sun.ts writes pubs.json atomically (temp + rename) to prevent truncation on crash

## Key Decisions
- **Geometric shadow projection** over raster — mathematically precise, 60fps capable
- **DSM - DTM** over ground estimation — eliminates expensive sliding window minimum
- **No framework** — Canvas 2D rendering, framework adds overhead for no benefit
- **Static site** — all data pre-computed, no server at runtime
- **OSM as sole pub source** — cleaner data than FSA, no duplicates, accurate building polygons
- **PMTiles on R2** — single archive with range requests, replaces 47k individual .pbf files
- **R2 for data** — all pipeline output served from R2 (free tier: 10 GB storage, 10M reads/month), zero data in Pages dist/
- **Split pub data** — slim index for SPA startup (~12.6 MB / ~2 MB gzip) covers list/search/filter/sort; full per-pub files (`pub/{slug}.json`, ~3 KB each) loaded on pub selection AND fetched directly by the `/pub/[slug]` Pages Function so cold starts don't have to parse the index
- **Height-dependent radius** — reduces tile data ~30% by excluding short distant buildings
- **Hybrid DTM for horizons** — 1m Defra DTM for close-range detail, 50m OS Terrain 50 for distant ridges up to 3km
- **Terrain shadow edge rendering** — half-plane shadow from ridge distance + horizon angle, replaces elevation-based building shadow inflation
- **boto3 for R2 uploads** — ThreadPoolExecutor with 16 workers, ~67 files/s (vs ~5/s with wrangler)
