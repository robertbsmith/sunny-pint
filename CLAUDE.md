# Sunny Pint

Find sunny beer garden seats. Real-time shadow maps for UK pubs using LiDAR elevation data, OSM building outlines, and geometric sun projection.

**Domain**: sunny-pint.co.uk

## Status: UK Expansion In Progress

The app is fully functional for Norwich with ~142 pubs. The data pipeline supports any UK area via `--area` flag. Full UK pipeline (33k pubs) is being run.

## Quick Start

Runs in a devcontainer. Open in VS Code with Dev Containers extension.

```bash
pnpm install          # Frontend dependencies
just dev              # Vite dev server (port 5173)
just build            # Production build → dist/
just pipeline         # Data pipeline (Norwich default)
just pipeline area=uk # Full UK pipeline
```

## Data Pipeline

Data files are NOT in the repo — they're generated by the pipeline. To regenerate:

1. Download `england-latest.osm.pbf` from [Geofabrik](https://download.geofabrik.de/europe/great-britain/england.html) → `data/`
2. Run `just pipeline` — runs merge_pubs, download_inspire, build_gpkg, measure_heights, match_plots, generate_tiles
3. LiDAR tiles download automatically from EA WCS during measure_heights
4. INSPIRE plot data downloads automatically via download_inspire

Output lands in `public/data/pubs.json` and `public/data/tiles/*.pbf`.

See `docs/DATA_PIPELINE.md` for full details.

## Architecture

**Static site** — no backend at runtime. All shadow computation is client-side at 60fps.

### Frontend (`src/`)
- **Vite 8** + TypeScript 6 + Tailwind 4
- **Canvas 2D** — porthole renderer, sun arc widget
- **SunCalc** — sun position
- **Lucide** — icons
- **VitePWA** — offline/installable

Key modules:
- `circle.ts` — porthole canvas (buildings, shadows, bezel, compass, pub sign)
- `sunarc.ts` — sun arc time picker with play animation
- `shadow.ts` — geometric shadow projection
- `buildings.ts` — building data loader (fetches z14 .pbf tiles)
- `publist.ts` — pub list with search and distance sorting
- `location.ts` — GPS + Nominatim geocoding
- `weather.ts` — Open-Meteo cloud cover
- `share.ts` — snapshot image generation + Web Share API
- `url.ts` — deep linking via URL params
- `icons.ts` — Lucide DOM icons

### Data Pipeline (`scripts/`)
- `merge_pubs.py` — OSM .pbf → pub list (`data/pubs_merged.json`)
- `download_inspire.py` — bulk download all INSPIRE GML files (318 local authorities)
- `match_plots.py` — INSPIRE plots + buildings → outdoor areas → `public/data/pubs.json`
- `build_gpkg.py` — OSM .pbf → GeoPackage with buildings (streaming, no OOM)
- `measure_heights.py` — LiDAR DSM/DTM → building heights
- `generate_tiles.py` — GeoPackage → individual z14 .pbf tile files
- `areas.py` — shared `--area` flag (norwich/bristol/london/uk)

### Deployment
- **Cloudflare Pages** — `pnpm build` then deploy `dist/`
- Everything in `dist/` is static — works on any host
- Cloudflare Pages does NOT support HTTP byte range requests (PMTiles won't work)

## Critical Technical Context

- Building heights use DSM minus DTM — simple subtraction, fast
- 90th percentile of LiDAR pixels within building footprint captures ridge height
- Shadow quads from overlapping buildings need offscreen canvas compositing to prevent opacity stacking
- Shadow max length capped at 200m to avoid infinite geometry near sunrise/sunset
- Height-dependent building filter: short buildings excluded if too far to cast useful shadows
- Day/night transition uses `dayFrac = (altitude + 2) / 10` for smooth 30-minute twilight
- CartoDB tile URL needs `rastertiles/` prefix: `rastertiles/voyager_labels_under/{z}/{x}/{y}.png`
- Porthole uses tile zoom 18 metres-per-pixel for building overlay alignment
- LiDAR sources: EA WCS (England), JNCC WCS (Scotland), NRW COG (Wales)
- Outdoor area polygons support holes (evenodd canvas fill) for enclosed buildings

## Key Decisions
- **Geometric shadow projection** over raster — mathematically precise, 60fps capable
- **DSM - DTM** over ground estimation — eliminates expensive sliding window minimum
- **No framework** — Canvas 2D rendering, framework adds overhead for no benefit
- **Static site** — all data pre-computed, no server at runtime
- **OSM as sole pub source** — cleaner data than FSA, no duplicates, accurate building polygons
- **Individual .pbf tiles** over PMTiles — Cloudflare Pages breaks PMTiles range requests
- **Height-dependent radius** — reduces tile data ~30% by excluding short distant buildings
