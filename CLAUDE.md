# Sunny Pint

Find sunny beer garden seats. Real-time shadow maps for UK pubs using LiDAR elevation data, OSM building outlines, and geometric sun projection.

**Domain**: sunny-pint.co.uk

## Status: UK Live (England complete, Scotland/Wales buildings ingested)

38k pubs (33k England + 2.4k Scotland + 2.5k Wales). England pubs have LiDAR heights, horizons, outdoor areas, and Sunny Ratings. Scotland/Wales have buildings but no LiDAR heights yet (needs measure_heights run). Data served from Cloudflare R2 via `data.sunny-pint.co.uk`.

## Quick Start

Runs in a devcontainer. Open in VS Code with Dev Containers extension.

```bash
pnpm install          # Frontend dependencies
just dev              # Vite dev server (port 5173)
just release          # Production build + SEO landing pages → dist/
just pipeline         # Data pipeline (Norwich default)
just pipeline area=uk # Full UK pipeline
```

## Data Pipeline

Data files are NOT in the repo — they're generated by the pipeline and served from R2. To regenerate:

1. Download OSM extracts from [Geofabrik](https://download.geofabrik.de/europe/united-kingdom/) → `data/`
   - `england-latest.osm.pbf`, `scotland-latest.osm.pbf`, `wales-latest.osm.pbf`
2. Run `just pipeline` — runs all pipeline steps (see justfile for order)
3. LiDAR data downloads automatically from Defra survey catalogue (5km bundles)
4. INSPIRE plot data downloads automatically via download_inspire
5. Run `just deploy-data` to upload results to R2

Output: `public/data/pubs.json` (local), `public/data/pubs-index.json` (R2), `public/data/detail/*.json` (R2), `public/data/buildings.pmtiles` (R2).

Pipeline v2 (`pipeline/run.py`) is built but untested — merges 9 scripts into 5 stages with manifest-based caching. See `pipeline/DESIGN.md`.

## Architecture

**Static site** — no backend at runtime. All shadow computation is client-side at 60fps.

### Frontend (`src/`)
- **Vite 8** + TypeScript 6 + Tailwind 4
- **Canvas 2D** — porthole renderer, sun arc widget
- **SunCalc** — sun position
- **Lucide** — icons
- **VitePWA** — offline/installable

Key modules (see `.claude/rules/frontend.md` for full list):
- `circle.ts` — porthole canvas (buildings, shadows, bezel, compass, pub sign)
- `sunarc.ts` — sun arc time picker with play animation
- `shadow.ts` — geometric shadow projection
- `buildings.ts` — building data loader (PMTiles range requests from R2)
- `publist.ts` — pub list with search and distance sorting
- `state.ts` — app state, pub selection, time

### Data Pipeline (`scripts/`)
- `merge_pubs.py` — OSM .pbf → pub list (`data/pubs_merged.json`)
- `download_inspire.py` — bulk download all INSPIRE GML files (318 local authorities)
- `build_inspire_gpkg.py` — index downloaded GML files into a spatial GeoPackage
- `build_gpkg.py` — OSM .pbf → GeoPackage with buildings (streaming, no OOM)
- `measure_heights.py` — LiDAR DSM/DTM → building heights
- `match_plots.py` — INSPIRE plots + buildings → outdoor areas → `public/data/pubs.json`
- `compute_horizons.py` — terrain horizon profiles from DTM for each pub
- `generate_tiles.py` — GeoPackage → PMTiles archive (buildings.pmtiles)
- `precompute_sun.ts` — simulate sun path on equinox → Sunny Rating per pub
- `generate_pages.ts` — build-time SEO landing pages (city, theme, sitemap)
- `areas.py` — shared `--area` flag (norwich/bristol/london/uk)

### Data Serving
- **Cloudflare R2** — bucket `sunny-pint-data`, domain `data.sunny-pint.co.uk`
  - `pubs-index.json` (~1.6 MB gzip) — slim pub list loaded at app startup
  - `detail/{lat}_{lng}.json` — per-pub heavy data (outdoor polygon, elev, horizon) in 0.1° grid chunks, loaded on pub selection
  - `buildings.pmtiles` — building vector tiles, range-requested by pmtiles.js
  - `og/{slug}.jpg` — pre-rendered OG card images (TODO: not yet generated)
- **Cloudflare Pages** — `dist/` contains only the SPA shell (HTML/JS/CSS/icons), zero data files
  - `just release` strips all data from dist/ before deploy
  - `just deploy-data` uploads data to R2 separately

### Deployment
- **Cloudflare Pages** — auto-deploys on git push to main
- Per-pub HTML pages rendered on-demand by Pages Functions (`functions/pub/[slug].ts`)
- OG images: pre-rendered as static JPEGs on R2 (planned), fallback to generic banner
- Static city/theme landing pages generated at build time by `generate_pages.ts`
- Pages Functions fetch pub data from R2 (not from dist/)

## Critical Technical Context

- Building heights use DSM minus DTM — simple subtraction, fast
- 90th percentile of LiDAR pixels within building footprint captures ridge height
- Shadow quads from overlapping buildings need offscreen canvas compositing to prevent opacity stacking
- Shadow max length capped at 200m to avoid infinite geometry near sunrise/sunset
- Height-dependent building filter: short buildings excluded if too far to cast useful shadows
- Day/night transition uses `dayFrac = (altitude + 2) / 10` for smooth 30-minute twilight
- Stadia Maps tile URL: `tiles.stadiamaps.com/tiles/osm_bright/{z}/{x}/{y}.png` (API key needed for server-side fetches, browser uses domain allowlisting)
- Porthole uses tile zoom 18 metres-per-pixel for building overlay alignment
- LiDAR sources: EA WCS (England), JNCC WCS (Scotland), NRW COG (Wales)
- Outdoor area polygons support holes (evenodd canvas fill) for enclosed buildings
- Buildings with `ground_elev = NULL` get `elev: 0` in tiles — shadow.ts assumes pub elevation for these (otherwise effectiveHeight goes negative and shadows vanish)
- precompute_sun.ts writes pubs.json atomically (temp + rename) to prevent truncation on crash

## Key Decisions
- **Geometric shadow projection** over raster — mathematically precise, 60fps capable
- **DSM - DTM** over ground estimation — eliminates expensive sliding window minimum
- **No framework** — Canvas 2D rendering, framework adds overhead for no benefit
- **Static site** — all data pre-computed, no server at runtime
- **OSM as sole pub source** — cleaner data than FSA, no duplicates, accurate building polygons
- **PMTiles on R2** — single archive with range requests, replaces 47k individual .pbf files (R2 supports range requests, Pages doesn't)
- **R2 for data** — all pipeline output served from R2 (free tier: 10 GB storage, 10M reads/month), zero data in Pages dist/
- **Split pub data** — slim index for app startup (~1.6 MB gzip), heavy fields loaded per-pub on selection via geographic chunks
- **Height-dependent radius** — reduces tile data ~30% by excluding short distant buildings
